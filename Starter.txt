# AppStack Bootstrap Protocol
# Follow these exact steps when creating a new portfolio application.

1. Provision Infrastructure
   - Create a new Supabase project (save URL, Anon Key, Service Role Key).
   - Create a new Vercel project (do not link to GitHub yet).

2. Initialize Local Environment
   - Open PowerShell.
   - `cd C:\Projects\appstack\_APPS\`
   - `mkdir NewAppName`
   - `cd NewAppName`
   - `npx create-next-app@latest . --name newappname` (use lowercase for npm compliance)

3. Next.js Setup Wizard Answers
   - TypeScript: Yes
   - ESLint: Yes
   - Tailwind: Yes
   - src/ directory: No
   - App Router: Yes
   - Custom alias: No

4. Install Baseline Dependencies
   - `npm install @supabase/supabase-js`

5. Establish Environment Variables
   - Create `.env.local`
   - Add Supabase credentials.

6. Version Control
   - Initialize isolated git repository if separating from the main AppStack monorepo.
   - Push initial baseline commit.

### Architecture
- Feature-based modular monolith (see `Schema.txt`, `Conventions.txt`).
- Domains: `features/marketing`, `features/territory`, `features/booking`,
  `features/billing`, `features/admin`.
- Supabase: `shared/lib/supabase.ts` -> `@/shared/lib/supabase`.

### Storefront / payment model (final)
- Each operator = shop manager of their storefront inside NeighborDogs.
- Walk payments separate from $149/month operator subscription.
- Walk payments: Stripe Connect Standard accounts + direct charges.
- Operator is merchant of record; operator pays Stripe fees; NeighborDogs $0 on walks.
- No application_fee_amount, transfer_data.destination, or on_behalf_of.
- Operators set prices in Admin (`operator_prices`); checkout re-reads server-side
  (inline `price_data` — not Stripe Dashboard Products/Prices for walks).
- NeighborDogs stores `profiles.stripe_connect_id` only — not operator banking info.
- Do not split Stripe accounts during the walk-payment fix (see 00_PICKUP.txt).
- CURRENT CODE WRONG: Express + destination charge — fix to Standard + direct charges.
