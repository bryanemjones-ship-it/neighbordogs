# AppStack Bootstrap Protocol
# Follow these exact steps when creating a new portfolio application.

1. Provision Infrastructure
   - Create a new Supabase project (save URL, Anon Key, Service Role Key).
   - Create a new Vercel project (do not link to GitHub yet).

2. Initialize Local Environment
   - Open PowerShell.
   - `cd C:\Projects\appstack\_APPS\`
   - `mkdir NewAppName`
   - `cd NewAppName`
   - `npx create-next-app@latest . --name newappname` (use lowercase for npm compliance)

3. Next.js Setup Wizard Answers
   - TypeScript: Yes
   - ESLint: Yes
   - Tailwind: Yes
   - src/ directory: No
   - App Router: Yes
   - Custom alias: No

4. Install Baseline Dependencies
   - `npm install @supabase/supabase-js`

5. Establish Environment Variables
   - Create `.env.local`
   - Add Supabase credentials.

6. Version Control
   - Initialize isolated git repository if separating from the main AppStack monorepo.
   - Push initial baseline commit.

|------|---------|

### Production data sources (returning customer booking path)
| Data | Table / field | Used by |
|------|----------------|---------|
| Returning customer | `public.customers` | customer-lookup API |
| Slot conflicts | `public.bookings` | slots API via fetchBookingsForDate |
| Operator slug/prices | `profiles.slug`, `operator_prices` | operator-resolve |
| Connect status | `profiles.stripe_connect_id` | walk checkout |

NOT production source of truth:
- `localStorage` key `rdgw_customers_v1` (fallback only in ClientBookingFlow)
- `test_demo_customers` / `test_flag` (admin/demo paths only)

### Verified production public.bookings columns
id, customer_id, walker_id, type, date, start_time, end_time, price_cents,
status, is_emergency, created_at, customer_name

NOT in production public.bookings:
blocked_end_time, test_flag

Slot generation: hardcoded operating slots (08:00-18:00, 30-min steps), filtered
by existing public.bookings using end_time as block end. public.walkers not required.

### Locked payment model
- Operators pay NeighborDogs $149/month for software access.
- Clients pay operators for walks through the app via Stripe Connect.
- Operators absorb Stripe card processing fees.
- NeighborDogs receives $0 from walk payments.
- No application_fee_amount; no STRIPE_WALK_PLATFORM_FEE_PERCENT.
See `OPERATOR_COMMERCE_RELEASE_CHECKLIST.txt` and `00_PICKUP.txt`.
