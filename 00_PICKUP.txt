DECISIONS (FINAL ONLY):
CLARIFIED STOREFRONT / PAYMENT MODEL:
- Each operator runs their own service storefront inside NeighborDogs; operator is
  shop manager.
- Operators manage walk/service prices in NeighborDogs Admin (operator_prices).
- Customers see operator current prices on public booking page (/book/[slug]).
- Checkout price is current operator price when customer submits order (server re-read
  at checkout; inline price_data — not Stripe Dashboard Products/Prices for walks).
- Walk payments are separate from the $149/month operator subscription.
- Walk payments use Stripe Connect Standard accounts and direct charges.
- Operator is merchant of record for walk payments; operator pays Stripe card
  processing fees; operator Stripe balance bears refunds and disputes.
- Stripe processes customer card on the operator connected account (direct charge).
- Operators onboard/connect directly with Stripe for payout/banking; NeighborDogs
  stores only stripe_connect_id/status, not banking information.
- Bryan / NeighborDogs receives $0 from walk payments — no platform percentage,
  commission, application fee, or cut. No application_fee_amount.
- No transfer_data.destination. No on_behalf_of. NeighborDogs is the software
  layer only.
- STRIPE_WALK_PLATFORM_FEE_PERCENT is not part of this release.
- Do not say payment happens outside the app or is handled directly with the walker.

STRIPE ACCOUNT SEPARATION (during current fix — not a split project yet):

- Keep the current payment fix to one variable: Standard Connect + direct charges
