# NeighborDogs Schema & Structure Reference
# Purpose: Verified production data paths, storefront payment model, codebase layout
# Updated: March 2026

---

## A. Storefront payment data flow

```
Operator Admin (PricingPanel)
  -> public.operator_prices (operator-controlled walk prices)

Public booking (/book/[slug])
  -> resolveOperatorBySlug (profiles.slug)
  -> read operator_prices for display

Customer submits order
  -> POST /api/booking/checkout
  -> walk-checkout.ts: quoteWalkPriceCents (server re-read operator_prices)
  -> Stripe Checkout: inline price_data (NOT Stripe Dashboard Product/Price IDs)
  -> Connect: on_behalf_of + transfer_data to profiles.stripe_connect_id
  -> NO application_fee_amount; NeighborDogs $0 on walks

Operator payout
  -> Stripe Connect onboarding only (POST /api/stripe/connect/onboard)
  -> NeighborDogs stores stripe_connect_id only; no banking fields
```

Display-price risk: `POST /api/booking/validate` may use DEFAULT_PRICES without operator
context — payment screen amount can diverge from operator_prices until fixed.

## B. Verified production tables (returning-customer booking path)

### public.customers
Returning customer lookup (Find My Profile).
- Used by: `app/api/booking/customer-lookup/route.ts`
- Key fields: id, email, name, approved
- NOT used for this path: test_demo_customers, test_flag

### public.bookings
Slot conflict reads and (intended) walk booking persistence.

**Verified production columns:**
- id
- customer_id
- walker_id
- type
- date
- start_time
- end_time
- price_cents
- status
- is_emergency
- created_at
- customer_name

**NOT in verified production public.bookings:**
- blocked_end_time
- test_flag

**Referenced by code but NOT verified in production** (investigate before checkout E2E):
- payment_status
- stripe_checkout_session_id
- stripe_payment_intent_id
- operator_id
- stripe_connect_account_id
- customer_email
- platform_fee_cents
- test_flag (insert path in walk-checkout)
- blocked_end_time (insert path in walk-checkout)

Slot loader (49c0de6): maps end_time as blocking end; no test_flag filter.

### public.operator_prices
Operator walk pricing — source of truth for storefront display and checkout quote.
- Used by: PricingPanel (write), operator-resolve + walk-checkout (read)
- north-hills example (cents): w20 2900, w30 3500, w60 4900, emergency 4900,
  buddyAddon 1000, weeklyDiscount 15

### profiles
Operator resolve and Connect.
- slug (e.g. north-hills)
- stripe_connect_id (required for walk card checkout)
- north-hills operator id: bb52d0f0-03c4-44b7-912f-b47350d71cd0

### public.walkers
NOT required for time-slot generation. Empty row count does not block slots.

## C. Slot generation (no DB table)

Hardcoded in `features/booking/lib/slots.ts`:
- OP_START 08:00, OP_END 18:00, SLOT_INCREMENT_MIN 30
- `generateOperatingSlots()` -> `filterAvailableSlots()` using public.bookings intervals
- 15-minute buffer in slotRangeForWalk

Flow: `GET app/api/booking/slots` -> `fetchBookingsForDate(date)` -> filter -> JSON

## F. Read-only investigation summary

**What matches storefront model:**
- Admin writes operator_prices; public booking reads them; checkout re-quotes server-side.
- Inline price_data; no application_fee_amount; Connect payout to operator.
- Connect onboarding UI + API exists.

**Known conflicts:**
1. validate route — operator context missing; DEFAULT_PRICES fallback.
2. walk-checkout.ts — insert columns may exceed verified production bookings.
3. walk-webhook.ts — update columns may exceed verified production bookings.
4. Same anti-drift class as pre-49c0de6 slot loader failure.

## G. Codebase structure (feature-based modular monolith)

```
app/
├── (marketing)/          /, /territory-preview
├── (client)/book/        /book, /book/[slug]
├── dashboard/            operator admin
├── login/
└── api/                  booking, admin, stripe, territory-preview

features/
├── marketing/
├── territory/
├── booking/              ClientBookingFlow, pickers, payment, db, slots, operator-prices
├── billing/              walk-checkout, walk-webhook, stripe-connect, stripe
└── admin/                AdminPanel + PricingPanel

shared/lib/supabase.ts    browser + createSupabaseServerClient()
_legacy/                  reference only — never import
```

Import alias: `@/*` -> project root.
