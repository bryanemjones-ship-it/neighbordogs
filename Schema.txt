# Purpose: Verified production data paths + codebase layout (March 2026)

## A. Verified production tables (returning-customer booking path)

### public.customers
Returning customer lookup (Find My Profile).
- Used by: `app/api/booking/customer-lookup/route.ts`
- Key fields: id, email, name, approved
- NOT used for this path: test_demo_customers, test_flag

### public.bookings
Slot conflict reads and walk booking persistence.
Verified production columns:
- id
- customer_id
- walker_id
- type
- date
- start_time
- end_time
- price_cents
- status
- is_emergency
- created_at
- customer_name

NOT in verified production public.bookings:
- blocked_end_time
- test_flag

Slot loader maps end_time as blocking end for overlap filtering after read.

### public.operator_prices
Operator walk pricing for /book/[slug] (server-side quote).
Known north-hills example (cents): w20 2900, w30 3500, w60 4900, emergency 4900,
buddyAddon 1000, weeklyDiscount 15.

### profiles
Operator resolve for /book/[slug].
- slug (e.g. north-hills)
- stripe_connect_id (required for walk card checkout to succeed)
Known north-hills operator id: bb52d0f0-03c4-44b7-912f-b47350d71cd0

### public.walkers
NOT required for time-slot generation. Empty row count does not block slots.

## B. Slot generation (no DB table)

Hardcoded in `features/booking/lib/slots.ts`:
- OP_START 08:00, OP_END 18:00, SLOT_INCREMENT_MIN 30
- `generateOperatingSlots()` produces base candidates
- `filterAvailableSlots()` removes overlaps using public.bookings intervals
- 15-minute buffer between appointments applied in slotRangeForWalk

Flow: `GET app/api/booking/slots` -> `fetchBookingsForDate(date)` -> filter -> JSON slots

## F. Codebase structure (feature-based modular monolith)

```
app/
├── (marketing)/          /, /territory-preview
├── (client)/book/        /book, /book/[slug]
├── dashboard/            operator admin
├── login/
└── api/                  booking, admin, stripe, territory-preview

features/
├── marketing/
├── territory/
├── booking/              ClientBookingFlow, pickers, payment, db, slots
└── admin/                AdminPanel + panels

shared/lib/supabase.ts    browser + createSupabaseServerClient()
_legacy/                  reference only — never import
```

Import alias: `@/*` -> project root.
Supabase: `@/shared/lib/supabase`.

### Domain -> route mapping
